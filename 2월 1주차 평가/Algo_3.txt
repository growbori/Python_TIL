3-1 완전검색에 대해 간단히 설명하고 장점과 단점을 간단히 설명하시오.

완전 탐색이란 알고리즘을 구하는 과정에서 발생할 수 있는 모든 경우의 수를 비교해보며 정답을 찾아가는 과정을 말한다.
완전 탐색을 통해 답을 구하면 모든 경우의 수를 확인하기 때문에 오류가 발생할 확률이 줄어든다는 장점을 가지고 있다.
하지만 탐색 시간이 너무 오래 걸린다(시간복잡도가 높다)는 단점을 가지고 있다.

(Run 은 연속된 숫자 3개, Triplet은 같은 숫자 3개인 경우를 말하고 Run과 Triplet의 개수가 2개 이상이면 baby-gin이다.)

그럼 c = [0, 0, 1, 1, 1, 0, 0, 0, 3, 0, 0, 0, 0] 이런식으로 정리 될 것이다.
